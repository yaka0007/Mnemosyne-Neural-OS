# Neural Coding Handbook

> **The definitive guide to building production software through human-AI collaboration — without a traditional development background.**

---

## What Is This Handbook?

This handbook documents **Neural Coding** — a discipline for building real, production-grade software systems through structured collaboration with AI, where the practitioner's primary skill is systems thinking and domain expertise, not programming language knowledge.

It is a living document. As the practice evolves, so does this handbook.

---

## Who This Is For

- Non-developers who want to build real software products
- Domain experts who have ideas they can't implement without a development team
- Product managers, architects, and technical leaders who want to understand the methodology
- Developers who work with AI and want to think more rigorously about their collaboration patterns
- Anyone who has been told "you need to learn to code first"

---

## Who This Is Not For

This handbook is not a beginner's guide to AI chatbots. It is not about prompt engineering as a productivity hack. It is not about getting AI to write code for you while you step away.

Neural Coding requires serious engagement with complex systems. It requires patience, intellectual honesty, and a willingness to hold a large vision in your head for a long time.

---

## The Proof of Concept

This handbook was written by the same person who built [Mnemosyne Neural OS](https://github.com/yaka0007/Mnemosyne-Neural-OS) — a 290,000-line, production-grade AI intelligence platform — using exactly the methodology described here.

No prior coding background. No development team. Eighteen months.

The numbers are real. The code is audited. The CI pipeline is green.
