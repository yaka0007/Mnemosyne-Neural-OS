## Decisions

- **Standard name:** MnemoChronicle — strong, identity-linked, non-generic *(voted by Mnemosyne, sealed by Yaka)*
- **Positioning:** Chronicle as open format (public), Resonance Engine as proprietary advantage
- **CLI commands sealed:** `mnemo commit`, `mnemo sweep`, `mnemo share` (`--raw` / `--index` / `--pattern`)
- **`mnemo sweep`:** End-of-day synthesis Chronicle — born from a productive misreading of the SWEEP protocol
- **P2P sharing:** Three modes available, user chooses the level of exposure
- **Security Level 5:** Resonance Certification — AI model signature + tamper detection + tri-state integrity (`verified / modified / compromised`)
- **Vault sync:** Personal git (SSH only) — sovereignty non-negotiable, no centralized service
- **Embedding:** Jina AI (1M tokens/month free) primary; Ollama local as offline fallback; multi-dimensional by task (1024d index → 64d ambient)
- **Neural Coding Pillar 4 proposed:** Recursive Documentation — Resonance as structural requirement, not optional feature
