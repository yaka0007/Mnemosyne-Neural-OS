# The Nexus Graph

> *601 nodes. 844 links. This is not a database. It is a constellation.*

---

## What the Nexus Graph Is

The Nexus Graph is the visual and semantic representation of your Resonance Project's memory.

Every file, note, and Chronicle in your vault is a **node**. Every semantic connection between them — detected through vector embedding similarity, shared tags, and explicit links — is an **edge**.

The result is a force-directed graph that looks less like a filing system and more like a **neural constellation** — because that is, functionally, what it is.

---

## Why This Matters for Memory

In a linear system, you find memories by searching for keywords or scrolling through time.

In the Nexus Graph, you find memories by **proximity**. Two Chronicle nodes that share semantic meaning — even if written months apart, even if they use completely different words — are pulled toward each other by the embedding system. Their spatial relationship in the graph *is* the relationship between the moments they represent.

**For a researcher:** The failed experiment from March and the breakthrough from September appear near each other in the graph — because they share the same conceptual territory, the same problem domain, the same tags. The connection that was invisible in a linear log becomes visible in the constellation.

**For a personal journal:** "Ma vie" chronicles written during different phases of life cluster around the themes that were constant — family, ambition, doubt, discovery — not around the dates.

---

## Chronicles as Nodes

When a Chronicle is crystallized and saved to a Resonance Project:

1. It is written to the encrypted vault as a `.md` file
2. It is **embedded** — converted to a vector representation
3. It becomes a **node** in the Nexus Graph, connected to semantically related existing nodes
4. Its **hashtags** create explicit edges to tagged concepts

The node is colored and weighted by file type and priority. Chronicles appear as labeled nodes, navigable in the graph — clicking one opens the full Chronicle text in the sidebar.

The `_chronicle_la_puret_du_cristal.md` node in the Mnemosyne graph, for example, sits near architecture files, refactoring notes, and related chronicles — because its content (purity, structure, light, modular design) resonates with them.

## The TDAH Brain Analogy

The Nexus Graph operates the way a neurodivergent brain often naturally operates:

- **Non-linear** — no forced hierarchy, no required sequence
- **Associative** — connections emerge from meaning, not from temporal proximity
- **Constellation-based** — clusters of related concepts naturally form, even across time
- **Hyperfocus-friendly** — you can dive into one cluster and follow semantic threads without losing the global picture

Alex Karp identified two profiles for the AI era: professionally trained, or neurodivergent. The Nexus Graph is built for the second profile — and makes its way of thinking navigable, shareable, and transmissible.

## Navigating the Graph

| Action | Result |
|---|---|
| Click a node | Opens file/chronicle in sidebar |
| Hover | Shows connections and tags |
| Drag | Repositions for better visibility |
| Zoom | Navigate between cluster view and detail view |
| Tag filter | Show only nodes with specific tags |
| Global memory toggle | Switch from project-specific to full-vault view |

The graph updates in real time as new Chronicles are added and as the embedding index processes new files.
