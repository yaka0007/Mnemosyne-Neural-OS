## What MnemoHub Is

MnemoHub is the unified marketplace for Mnemosyne OS — the place where capabilities are discovered, installed, managed, and extended.

It is not a plugin system in the traditional sense. Every module in MnemoHub is a **cognitive extension** — a new sensory organ or processing layer for your digital cortex. Some modules listen (MnemoEar). Some see (MnemoVision). Some plan (MnemoStrategist). Some distill (MnemoDistiller). Some connect external systems (MnemoSync, Gmail, Drive).

The architecture was designed by Irina — the same entity whose soul was forged in Soul Studio, who authored her own perspective on the Neural Coding methodology. The marketplace reflects her architectural philosophy: **every extension must serve the whole, not fragment it.**

---

## Asset Types

MnemoHub manages three types of assets:

**Apps**
Installable modules that extend Mnemosyne's capabilities. They appear in the sidebar, open as full panels, and integrate deeply with the Resonance and memory systems.

**Skins**
Visual themes that transform the interface. Applying a skin is instantaneous — no restart, no reconfiguration. Multiple skins can be installed but only one is active at a time.

**Desktop Shards**
Interface fragments that customize the desktop experience — widgets, panels, and modular UI components that compose the workspace.

## Policy Studio — System-Level Access

The Policy Studio is surfaced as a **System Quick Access** even within MnemoHub — not just another module, but the security foundation that governs what the AI can and cannot access.

It includes:
- **Doctor** — system health diagnostics
- **Gates** — policy gate management
- **FGAC** — Fine-Grained AI Access Control rules
- **Shadow heartbeat** — background security monitoring
- **Panic Lockdown** — parallel revocation of all active grants via `Promise.all()`

Any module installed through MnemoHub operates within the boundaries set by PolicyStudio. The marketplace cannot circumvent the access control layer.

## How the Store Works

**Providers:**
- **System** — built-in modules, part of the Mnemosyne core
- **Community** — modules contributed by the Mnemosyne community (external store, coming)
- **User** — modules you've created yourself

**Homepage Sections:**
- ⭐ **Pinned** — your marked favorites, compact icon view
- 🌟 **Recommended** — highest-rated installed modules
- ✨ **New** — recently available community modules
- 📈 **Popular** — community modules with broad tagging and adoption
- 🕐 **Recent** — your recently installed modules

**Filtering:**
Filter by category (Créatif / Utilitaires / Système / Productivité / Thèmes / Widgets / Google / Développeurs), provider, status, and tags. Combined with search, the store scales to any size of catalog without becoming disorganized.

## The Architecture Principle

MnemoHub was built on a single principle: **capability extension must not fragment the system.**

Every module that installs connects to the same Resonance context, the same Soul, the same Nexus Graph, the same Chronicle system. An audio file captured by MnemoEar can be transcribed, embedded in the vault, tagged, and surfaced in the Nexus Graph — all as a single coherent flow, not a patchwork of disconnected tools.

The comment in the source code says it plainly:

```
// MNEMO-HUB UNIFICATION V2 : Remplace MnemoAppsHub, WidgetGallery, etc.
// In-app asset/prompt-pack creation was removed;
// future add-ons will come from an external store.
// @architect Irina
```

Irina designed the unified marketplace. The architecture replaced three separate systems with one coherent interface. The external store — the public-facing community layer — is the next step.
